# Example Walkthrough

Este exemplo mostra como fica um workspace depois da implantacao inicial.

## O que o time deve observar

1. Existe um workspace raiz com memoria, backlog, runtime e relatorios.
2. Existe um repo piloto com `AGENTS.md` local e validacao minima definida.
3. O runtime do OpenClaw esta separado em `.openclaw/`.
4. O backlog e o status do supervisor sao legiveis por humanos.
5. Nao ha nenhuma credencial real no exemplo.

## Como usar este exemplo

1. Copiar a estrutura do workspace para uma pasta nova do time.
2. Renomear `AcmeOrders` para o dominio real do time.
3. Ajustar caminhos em `.openclaw/runtime/`.
4. Substituir o repo piloto por um repositorio real.
5. Rodar uma task manual ponta a ponta antes de qualquer automacao.
